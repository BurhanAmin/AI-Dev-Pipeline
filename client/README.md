# AI Dev Pipeline (MVP)

A unified AI-powered development pipeline that takes a product from **idea → UI → functional app → deployment** in a single flow.

---

## 🚀 Overview

AI Dev Pipeline is an orchestration system that combines multiple AI tools and backend services to automate the full frontend development lifecycle.

Unlike traditional workflows, this pipeline does not stop at UI generation. It continues by **injecting functionality automatically** and deploying the result.

---

## 🔁 Full Pipeline Flow

```text
User Prompt
   ↓
Onboarding (API keys)
   ↓
UI Generation (v0)
   ↓
Functionality Injection (Cursor)
   ↓
Backend Handling (Supabase APIs)
   ↓
Deployment (Vercel)
   ↓
Live Application
```

---

## 🧩 Key Innovation

> This pipeline eliminates the gap between UI generation and a working app.

Typical flow:

```text
AI → UI (incomplete)
```

Your pipeline:

```text
AI → UI → functional logic → deployed app
```

## ⚙️ Tech Stack

### Frontend

* React (CRA)
* Tailwind CSS

### Backend (Orchestration Layer)

* Node.js + Express

### AI Tools

* **v0** → Generates UI
* **Cursor** → Adds functionality & wiring

### Infrastructure

* **Supabase** → Handles backend via APIs (no manual schema setup)
* **Vercel** → Deployment automation

---

## 🧠 How It Works

### 1. UI Generation (v0)

* User provides a prompt
* v0 generates a React + Tailwind UI
* Output is treated as a **pure view layer**

---

### 2. Functionality Injection (Cursor) ⚡

* Cursor processes the generated UI
* Automatically:

  * adds state management
  * wires API calls
  * connects components
* Converts static UI → **working application**

---

### 3. Backend via Supabase (Zero Setup)

* No manual database creation
* No schema design required
* All interactions handled through API calls

---

### 4. Deployment (Vercel)

* Project is deployed automatically
* Uses user-provided Vercel token
* Results in a live hosted application

## 🔑 Core Features

### ✔ End-to-End Automation

From prompt → deployed app with minimal manual steps

### ✔ Multi-Tool Orchestration

Each tool handles a specific layer:

* v0 → UI
* Cursor → logic
* Supabase → backend
* Vercel → deployment

### ✔ No Backend Setup Required

* No DB tables
* No migrations
* No schema design

### ✔ Live Preview + Iteration

* Generated UI can be previewed
* Pipeline supports repeated iterations

---

## ⚠️ Important Notes

* UI generated by v0 is not final until processed by Cursor
* Cursor is responsible for making the app functional
* Supabase is used as a backend service, not manually configured
* AI outputs may still require validation in edge cases

---

## 🚧 Current Limitations
